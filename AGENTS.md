## Architecture

This is a Node.js CLI for the Aliyun Yunxiao (云效) API, built with TypeScript and Commander.js.

### Entry Point

`src/index.ts` bootstraps the CLI with global options (`--verbose`, `--debug`) and registers command groups. The compiled entry point is `bin/yunxiao`, which checks Node.js >= 18 before requiring `dist/index.js`.

### Command Structure

Commands are organized into groups under `src/commands/`. Each group exports a factory function `createXxxCommand()` that returns a Commander `Command` with subcommands:

- `auth/` - login, logout, status
- `workitem/` - list, view, create, update (alias `wi`)
- `comment/` - add, list
- `config/` - get, set, list, path
- `pipeline/` - list, view, update, history, run, runs, run-view, run-latest, group-list, group-view, group-pipelines (alias `pl`)

Subcommand files follow the pattern of defining options, validating inputs, calling `getAuthenticatedClient()`, invoking the API client, and printing formatted output.

### API Client

`src/lib/yunxiao-client.ts` exports `YunxiaoApiClient` and `createYunxiaoClient()`. Important:

- Workitem and comment APIs use the `/projex` base path.
- Pipeline APIs use the `/flow` base path, derived by `getFlowBaseURL()` which replaces `/projex` with `/flow`.
- Pagination metadata is read from response headers (`x-page`, `x-per-page`, `x-total`, `x-total-pages`, `x-next-page`, `x-prev-page`).

Commands should use `getAuthenticatedClient()` from `src/utils/auth.ts` rather than `createYunxiaoClient()`, because it reads credentials from the config store.

### Configuration & Auth

`src/utils/config.ts` uses the `conf` package with `projectName: 'yunxiao-cli'`. It stores:

- `auth.access_token`, `auth.organization_id`, `auth.base_url`
- `defaults.*` including `organization_id`, `project_id`, `space_id`, `output_format`, `editor`
- `aliases.*` for command aliases (e.g. `wi` -> `workitem`)

`src/utils/auth.ts` provides `getAuthenticatedClient()`, `login()`, `logout()`, and `showAuthStatus()`. If the user is not authenticated, commands should exit with code 1.

### Output Formatting

`src/formatters/` contains output formatters used by commands:

- `table.ts` - `cli-table3` based human-readable tables
- `json.ts` - JSON output
- `csv.ts` - CSV output
- `markdown.ts` - Markdown output

Commands read `--output` or `--json` options and dispatch to the appropriate formatter.

### Logging

`src/utils/logger.ts` provides a singleton `logger` with levels `debug`, `verbose`, `info`, `warn`, `error`. The global `--verbose` and `--debug` options adjust the level in `src/index.ts`. Use `logger.success()` for positive feedback and `logger.error()` followed by `process.exit(1)` for fatal errors.

### Types

- `src/types/cli.ts` - CLI option interfaces and `OutputFormat`/`LogLevel` types.
- `src/types/yunxiao.ts` - API request/response interfaces.

## Development Notes

- Node.js >= 18 is required (enforced in `bin/yunxiao`).
- Prettier is configured with `singleQuote: true`, `trailingComma: 'es5'`, `printWidth: 100`.
- ESLint uses `@typescript-eslint/recommended` with `no-unused-vars` ignoring `_` prefixed args.
- Tests use Jest with `ts-jest`. Coverage thresholds are 70% for branches, functions, lines, and statements.
- The project follows Conventional Commits.
- `dist/` is gitignored but built during `npm run build` and `prepack`.
- Shell completion scripts in `completions/` are generated by `scripts/generate-completions.js`.
